# 回归测试用例（每次功能改动后必跑）

> 目标：确保跨视图导航、搜索、跳转、上下文清理在改动后保持可用。
> 环境：本地 HTTP 服务（例如 `python3 -m http.server 8888`），浏览器访问 `http://localhost:8888`。

## TC-01 页面基础加载

- 步骤：打开首页，等待加载完成。
- 期望：页面无白屏；顶部导航、主内容区、侧边栏可见；Console 无报错。

## TC-02 五个视图切换

- 步骤：依次点击“关系图谱 / 家族谱系 / 人物名录 / 回目视图 / 知识库”。
- 期望：每个视图都能正常渲染；切换后仅当前视图面板可见；无卡死。

## TC-03 图谱搜索（侧栏搜索）

- 步骤：在图谱侧栏搜索输入“宝玉”，点击一条结果。
- 期望：结果列表正确出现；点击后进入对应人物上下文；“清空上下文”可用。

## TC-04 树视图与名录搜索

- 步骤：切换到家族谱系，搜索“宝玉”；再切换到人物名录，搜索“黛玉”。
- 期望：两处搜索都能得到有效过滤结果，列表数量或内容发生预期变化。

## TC-05 视图内跳转

- 步骤：在人物卡中触发“查看关系”或“知识相关”入口；再使用返回按钮。
- 期望：能跳到目标上下文；返回可回到上一个可理解状态；不会丢失当前视图。

## TC-06 清空上下文

- 步骤：先选中任意人物或专题，再点击“清空上下文”。
- 期望：选中状态被清理；弹层关闭；图谱视图下回到重要人物概览；无异常提示。

## TC-07 关系对比

- 步骤：在“人物对比”选择两个人物并点击“查看关系”。
- 期望：关系视图/抽屉内容正确展示；二者关系网络可见；可继续进行其他操作。

## TC-08 移动端宽度回归（可选但建议）

- 步骤：将视口调整为约 390x844 后重复 TC-02 与 TC-03。
- 期望：视图可用；搜索可交互；无关键按钮遮挡。
